# プロジェクト概要

## 1. このプロジェクトの目的

Poker Timer App は、ポーカーサークルや小規模トーナメント運営で使うことを想定した Web / PWA のトーナメントタイマーである。

本アプリが解決したい課題は次のとおり。

- レベルとブレイクを含むトーナメント構造を、明示的なデータとして扱いたい
- 毎回同じ構造を手入力せず、プリセットとして保存・再利用したい
- 実行中のタイマー表示を安定させたい
- 将来的に Editor、保存、通知などの機能を拡張しても破綻しにくい設計にしたい

## 2. 現時点で確立している設計方針

本プロジェクトでは、次の方針を採用する。

- タイマー構造の正規形は `TimerStructure` とする
- Level と Break は同列の `TimerItem` として扱う
- タイマー進行は deterministic timer とし、epoch ms 基準で整合を取る
- UI は Usecase が生成した Snapshot を描画する
- Editor は `baseStructure + operations` で構造変更を表現する
- Preset は実行状態ではなく構造そのものを保存する

## 3. 何を目指しているか

このプロジェクトが目指しているのは、単なるカウントダウンタイマーではなく、**構造を作る・保存する・読み込む・実行する** を一貫して扱える実用トーナメントタイマーである。

短期的には次を重視する。

- タイマー挙動の安定化
- Editor の操作仕様確定
- Preset 管理の実用化
- docs の整備
- テストしやすい設計の維持

中長期的には次が候補になる。

- 設定画面の詳細化
- 構造 validation
- 音通知
- export / import
- サーバ保存
- 共有や同期

## 4. 非目標

現時点では、次は対象外または未確定とする。

- マルチユーザ同期
- 権限制御
- サークル分離
- トーナメント履歴の高度管理
- チップ管理や会計連携

これらは `docs/09_roadmap/roadmap.md` で扱う。
