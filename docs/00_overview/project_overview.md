# Project Overview

## 1. このプロジェクトは何か

Poker Timer App は、ポーカーサークルや小規模トーナメント運営のための Web タイマーアプリである。  
目的は「時間を数えること」だけではなく、**トーナメント構造を定義し、編集し、保存し、再利用できること**にある。

一般的な簡易タイマーでは、ブラインド情報や休憩の扱いが実装依存で曖昧になりやすい。  
本プロジェクトでは、運営で実際に必要になる以下をきちんとモデル化する。

- Level
- Break
- 現在進行中 item
- 次 item
- プリセット保存
- 構造の編集
- 実時間基準のタイマー整合

---

## 2. なぜこの設計を採用するのか

このプロジェクトでは、場当たり的な UI 実装ではなく、**仕様ベース設計**を採用している。  
先に「トーナメント進行の真実」を定義し、その上に UI・保存・操作を乗せる。

理由は次のとおり。

1. タイマーアプリは見た目より状態遷移が重要
2. Break のような例外扱いしがちな要素が後で破綻しやすい
3. 将来的に構造保存、共有、サーバ保存、同期などへ拡張したい
4. UI の変更にドメインやタイマーエンジンが巻き込まれないようにしたい

---

## 3. 現在の到達点

現時点では、以下が成立している。

- item 基準の TournamentStructure
- deterministic timer
- Editor による structure 編集
- Preset 保存 / 読込 / Rename / Delete
- Break を含む手動 / 自動遷移
- Snapshot ベースの UI

これは「モック」ではあるが、設計的にはかなり完成度が高く、今後の拡張の土台として十分に機能する。

---

## 4. 何をやろうとしているのか

このプロジェクトが目指すのは、単一用途の小さなタイマーではなく、**運営現場で実際に使える、構造化されたトーナメントタイマー**である。

短期的には次を目指す。

- タイマー実行の安定化
- Editor の実用性向上
- Preset 管理の実用化
- ドキュメント整備
- テスト容易性向上

中長期的には次を視野に入れる。

- 設定画面の具体化
- アラート / 音声通知
- 構造 validation
- JSON export / import
- サーバ保存
- マルチユーザや共有

---

## 5. 非目標

現時点では以下は未確定または対象外。

- マルチユーザリアルタイム同期
- 権限管理
- サークル分離
- トーナメント履歴管理
- 決済や会計との連携
- 複雑な通知戦略

これらは `docs/09_roadmap` で扱う。
